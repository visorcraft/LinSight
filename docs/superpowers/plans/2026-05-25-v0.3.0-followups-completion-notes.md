### Genuinely still open


* **Professional icon assets** — current PNGs are an in-house
  placeholder, not commissioned design. Replace before any
  marketing-grade screenshot. (Design work, not engineering.)
* **Human validation of the canvas editor's drag/resize ergonomics**
  — the layout renders and `save_layout` writes JSON. The audit-driven
  drag rewrite (PaletteRow now uses pure `Drag.Automatic`, no longer
  mixing with `MouseArea.drag.target`; tile bounds clamped during
  move) should make this materially smoother, but a real user still
  hasn't built a dashboard with it.
* **Tunnel `--allowed-cn <pattern>` filter** — `WebPkiClientVerifier`
  currently accepts any cert chained to the configured CA bundle, so
  the CA is a full-daemon-access trust boundary. A per-cert CN/SAN
  filter would tighten this for multi-tenant environments. Documented
  in `apps/linsight-tunnel/README.md` and `docs/security.md`.
* **Single source-of-truth icon generation script** — `packaging/icons/`
  and `apps/linsight-gui/resources/` are maintained as two separate
  rasterizations of the same SVG; a generation script would prevent
  drift on the next icon update.
* **`RPluginCtx.sysroot_set: u8` → stabby `Option<SString>`** — deferred
  to ABI v3; would be a wire-format break. The current
  bool-as-byte pairing is fine but non-idiomatic.
* **Daemon-side env-var status RPC** — `OverviewModel.envIsSet` reads
  the GUI process's env, which matches the daemon's only when both
  are launched together (e.g. systemd unit). A new daemon endpoint
  exposing its actual env state would handle the detached-daemon
  case.
* **GUI plugin first-launch acknowledgement dialog** — listed in
  `docs/security.md` as planned; lists third-party plugins from
  `$XDG_DATA_HOME/linsight/plugins/` before they're loaded for the
  first time.

### Misdiagnoses we caught and reverted

* A short-lived `Timer → tick()` Q_INVOKABLE pull in `OverviewModel`,
  plus a `docs/upstream/cxx-qt-binding-bug.md` writeup, claimed that
  cxx-qt 0.8's `qt_thread.queue()` setter calls didn't re-trigger
  QML binding evaluation. They actually do — the "stale value"
  symptom was 100% the Wayland compositor's stale-surface cache for
  unfocused windows, the same problem the in-app screenshot path
  fixes. A/B'd via env-var pivot: both paths produced live values
  in `--screenshot` captures (CPU 8.1 % vs CPU 6.6 % from
  back-to-back runs). The Timer pull was reverted; the writeup was
  deleted before filing upstream. Lesson: when an iteration tool
  contradicts in-memory diagnostics, trust the tool that bypasses
  the suspect path.
